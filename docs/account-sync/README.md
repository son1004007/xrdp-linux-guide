# 중앙 계정 관리 및 서버 계정 동기화 방안

## 1. 목적

빅데이터사업부 근무자 정보를 기준으로 사용자 계정을 1개의 중앙 서버에서 관리하고, 여러 서버에 동일한 계정 체계를 적용하기 위한 방안을 정리합니다.

적용 대상은 다음과 같습니다.

- Red Hat 계열 Linux
- Debian 계열 Linux
- Windows 11
- XRDP 접속용 Linux 서버
- 개발 서버, 분석 서버, 파일 서버 등 내부 업무 서버

다우오피스 그룹웨어 직접 연동은 관리자 권한이 없으므로 제외합니다. 대신 근무자 정보를 CSV 파일로 수집하여 중앙 계정 서버에 업로드하고, 해당 데이터를 기준으로 각 서버에 계정을 동기화하는 방식을 사용합니다.

---

## 2. 기본 방향

```text
근무자 정보 수집
      ↓
CSV 파일 작성
      ↓
중앙 계정 관리 서버 업로드
      ↓
계정 검증 및 표준화
      ↓
Linux / Windows 계정 동기화
```

---

## 3. 권장 구성

| 구분 | 권장 방식 |
|---|---|
| 계정 원본 데이터 | CSV 파일 |
| 중앙 계정 서버 | Rocky Linux 또는 Ubuntu Server |
| Linux 계정 관리 | FreeIPA 또는 LDAP + SSSD |
| 단순 MVP | CSV 기반 useradd 동기화 스크립트 |
| Windows 11 계정 관리 | Samba AD 또는 로컬 계정 동기화 스크립트 |
| 배치 실행 | systemd timer 또는 cron |
| 감사 로그 | 계정 생성, 변경, 비활성화 이력 저장 |

---

## 4. 단계별 도입 전략

### 1단계. CSV 기반 계정 동기화

가장 빠르게 적용 가능한 방식입니다.

- 근무자 이름, 이메일 수집
- 이메일 기반 계정 ID 생성
- 중앙 서버에 CSV 업로드
- 각 Linux 서버에서 동기화 스크립트 실행
- 필요한 사용자 계정 생성
- 퇴사자 또는 제외 대상 계정 비활성화

초기에는 이 방식으로 충분합니다.

### 2단계. FreeIPA 기반 중앙 인증

Linux 서버가 많아지면 FreeIPA 기반 중앙 인증으로 전환합니다.

- 사용자 계정 중앙 관리
- 비밀번호 정책 중앙 관리
- SSH Key 중앙 관리
- sudo 권한 중앙 관리
- Red Hat 계열과 Debian 계열 Linux 모두 SSSD 기반 연동

### 3단계. Windows 11 연동

Windows 11까지 계정 통합이 필요하면 Samba AD 구성을 검토합니다.

- Windows 11 도메인 조인
- Samba AD 기반 사용자 관리
- Linux는 SSSD로 AD 또는 LDAP 연동

다만 Windows까지 포함하면 운영 난이도가 높아지므로 Linux 계정 통합을 먼저 완료한 뒤 진행합니다.

## 6. 결론

현재 조건에서는 다우오피스 직접 연동보다 CSV 기반 중앙 계정 관리가 현실적입니다.

우선순위는 다음과 같습니다.

1. CSV 기반 계정 원본 데이터 관리
2. Linux 서버 계정 동기화 자동화
3. FreeIPA 기반 중앙 인증 전환
4. Windows 11은 필요 시 Samba AD로 확장

초기 목표는 모든 시스템을 한 번에 통합하는 것이 아니라, 계정 생성과 비활성화를 중앙에서 통제할 수 있는 구조를 먼저 만드는 것입니다.
