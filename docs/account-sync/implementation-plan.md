# 중앙 계정 관리 구축 작업계획서

## 1. 작업 목적

빅데이터사업부 근무자 계정을 1개의 중앙 서버에서 관리하고, 여러 Linux 서버와 Windows 11 환경에 일관된 계정 체계를 적용합니다.

초기 목표는 계정 생성, 비활성화, 권한 부여를 자동화하는 것입니다. 이후 필요 시 FreeIPA 또는 Samba AD 기반 중앙 인증 구조로 확장합니다.

---

## 2. 추진 배경

현재 서버별로 계정을 개별 생성하면 다음 문제가 발생합니다.

- 서버별 계정 생성 누락
- 퇴사자 또는 전출자 계정 잔존
- sudo 권한 관리 분산
- XRDP, SSH 접속 권한 관리 불일치
- 계정 현황 확인과 감사 대응 어려움

이를 개선하기 위해 사용자 기준 데이터를 중앙에서 관리하고, 각 서버는 해당 데이터를 기준으로 계정을 동기화합니다.

---

## 3. 적용 범위

## 포함 범위

- 빅데이터사업부 근무자 정보 수집
- 사용자 CSV 파일 표준 정의
- 중앙 계정 관리 서버 구성
- Linux 계정 생성 및 비활성화 자동화
- 관리자 그룹과 일반 사용자 그룹 분리
- 계정 동기화 로그 관리
- 운영 절차 문서화

## 제외 범위

- 다우오피스 직접 연동
- 그룹웨어 관리자 권한 기반 API 연동
- 인사 시스템 직접 연동
- 전사 AD 정책 변경
- 기존 업무망 보안 정책 변경

---

## 4. 목표 시스템 구성

```text
근무자 정보 CSV
      ↓
중앙 계정 관리 서버
      ↓
동기화 스크립트
      ↓
Linux 서버 / Windows 11
```

초기에는 CSV 파일과 스크립트 기반으로 구성합니다. Linux 서버 수가 늘거나 보안 요구사항이 강화되면 FreeIPA 기반 중앙 인증 구조로 확장합니다.

## 6. 1단계 상세 작업

## 6.1 사용자 CSV 양식 정의

기본 양식은 다음과 같습니다.

```csv
name,email,department,status,role
홍길동,hong@example.com,빅데이터사업부,active,user
김관리,admin@example.com,빅데이터사업부,active,admin
이퇴사,leave@example.com,빅데이터사업부,inactive,user
```

## 6.2 계정명 규칙

| 기준 | 규칙 |
|---|---|
| 기본 계정명 | 이메일 앞부분 사용 |
| 허용 문자 | 영문 소문자, 숫자, 하이픈, 언더스코어 |
| 금지 계정 | root, admin, test, oracle 등 예약 계정 |
| 중복 처리 | 중복 시 수동 검토 |

예:

```text
hong@example.com → hong
```

## 6.3 계정 상태 규칙

| status | 처리 |
|---|---|
| active | 계정 생성 또는 활성화 |
| inactive | 계정 잠금 및 로그인 차단 |

퇴사자 계정은 즉시 삭제하지 않고 잠금 처리합니다. 삭제는 데이터 백업과 소유 파일 이관 후 별도 처리합니다.

## 7. 2단계 상세 작업

## 7.1 중앙 서버 디렉터리 구조

```text
/opt/account-sync
├── data
│   └── users.csv
├── logs
│   └── sync.log
├── scripts
│   ├── sync-linux-users.sh
│   └── validate-users.py
└── README.md
```

## 7.2 권한 기준

```bash
sudo mkdir -p /opt/account-sync/{data,logs,scripts}
sudo chown -R root:root /opt/account-sync
sudo chmod 750 /opt/account-sync
sudo chmod 640 /opt/account-sync/data/users.csv
```

사용자 원본 파일은 일반 사용자가 수정할 수 없도록 제한합니다.

## 8. 3단계 상세 작업

Linux 동기화 스크립트는 다음 처리를 수행합니다.

1. CSV 파일 읽기
2. 계정명 생성
3. 기존 계정 존재 여부 확인
4. 신규 계정 생성
5. role 값에 따라 그룹 부여
6. inactive 계정 잠금
7. 처리 결과 로그 기록

예상 명령어는 다음과 같습니다.

```bash
sudo useradd -m -s /bin/bash username
sudo usermod -aG sudo username
sudo passwd -l username
```

Red Hat 계열에서는 관리자 그룹을 `wheel`로 사용합니다.

```bash
sudo usermod -aG wheel username
```

Debian 계열에서는 관리자 그룹을 `sudo`로 사용합니다.

```bash
sudo usermod -aG sudo username
```

## 10. 운영 전환 기준

다음 조건을 만족하면 운영 적용을 검토합니다.

- 테스트 서버 2대 이상에서 정상 동작
- 신규 계정 생성 정상
- inactive 계정 잠금 정상
- 관리자 그룹 부여 정상
- 롤백 절차 확인
- 중앙 CSV 파일 백업 절차 확인

---

## 11. 롤백 방안

동기화 오류 발생 시 다음 순서로 복구합니다.

1. 동기화 로그 확인
2. 변경된 계정 목록 확인
3. 신규 생성 계정 잠금
4. 관리자 그룹 부여 내역 회수
5. 이전 CSV 파일로 복원
6. 재동기화 또는 수동 복구

계정 삭제는 자동화하지 않습니다. 실수로 인한 홈 디렉터리 삭제를 방지하기 위해 비활성화까지만 자동 처리합니다.

## 12. 향후 확장 계획

## FreeIPA 전환

Linux 서버가 증가하면 FreeIPA로 전환합니다.

주요 전환 항목은 다음과 같습니다.

- 계정 중앙 인증
- SSH Key 중앙 관리
- sudo 정책 중앙 관리
- 비밀번호 정책 중앙 관리
- 서버별 로컬 계정 최소화

## Windows 11 연동

Windows 11까지 통합이 필요하면 Samba AD 구성을 검토합니다.

초기에는 Windows 로컬 계정 생성 스크립트를 사용할 수 있으나, 장기적으로는 도메인 기반 관리가 더 적절합니다.

## 14. 결론

현실적인 1차 목표는 완전한 통합 인증이 아니라 계정 생성과 비활성화의 중앙 통제입니다.

따라서 CSV 기반 동기화 구조로 빠르게 시작하고, Linux 서버 운영 규모가 커지면 FreeIPA 기반 중앙 인증으로 전환하는 방식이 적절합니다.
